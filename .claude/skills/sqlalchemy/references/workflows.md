## Initial Database Setup

### 1. Install Dependencies

```bash
cd toolbox/backend
pip install sqlalchemy[asyncio] asyncpg alembic
```

**Driver Choice:** `asyncpg` for async PostgreSQL (required for FastAPI). Do NOT use `psycopg2` (synchronous).

### 2. Configure Environment Variables

```bash
# .env or environment
DATABASE_URL=postgresql+asyncpg://postgres:password@localhost:5432/toolbox
DATABASE_ENABLED=true
```

**URL Format:** `postgresql+asyncpg://user:pass@host:port/dbname`

### 3. Create Engine and Session Factory

```python
# toolbox/backend/app/core/database.py
from sqlalchemy.ext.asyncio import create_async_engine, AsyncSession, async_sessionmaker
from app.core.config import settings

engine = create_async_engine(
    settings.database_url,
    echo=settings.debug,
    pool_pre_ping=True,
)

AsyncSessionLocal = async_sessionmaker(
    engine,
    class_=AsyncSession,
    expire_on_commit=False,
)
```

### 4. Define Models

```python
# toolbox/backend/app/models/db_models.py
from sqlalchemy import Column, String, DateTime
from sqlalchemy.ext.declarative import declarative_base
from datetime import datetime

Base = declarative_base()

class Job(Base):
    __tablename__ = "jobs"
    
    id = Column(String, primary_key=True)
    user_email = Column(String, nullable=False, index=True)
    tool_name = Column(String, nullable=False)
    status = Column(String, default="pending")
    created_at = Column(DateTime, default=datetime.utcnow)
```

### 5. Create Tables

```python
# toolbox/backend/app/main.py
from app.core.database import engine
from app.models.db_models import Base

@app.on_event("startup")
async def startup():
    async with engine.begin() as conn:
        await conn.run_sync(Base.metadata.create_all)
```

**For Production:** Use Alembic migrations instead of `create_all` (see Schema Migrations section).

## Schema Migrations with Alembic

### Initialize Alembic

```bash
cd toolbox/backend
alembic init alembic
```

This creates `alembic/` directory and `alembic.ini`.

### Configure Alembic for Async

Edit `alembic/env.py`:

```python
from app.core.database import engine
from app.models.db_models import Base

# Set target metadata
target_metadata = Base.metadata

# Use async engine
from sqlalchemy import pool
from sqlalchemy.ext.asyncio import async_engine_from_config

def run_migrations_online():
    connectable = async_engine_from_config(
        config.get_section(config.config_ini_section),
        prefix="sqlalchemy.",
        poolclass=pool.NullPool,
    )

    async with connectable.connect() as connection:
        await connection.run_sync(do_run_migrations)

def do_run_migrations(connection):
    context.configure(connection=connection, target_metadata=target_metadata)
    with context.begin_transaction():
        context.run_migrations()
```

### Create Migration

```bash
# Auto-generate migration from model changes
alembic revision --autogenerate -m "Add jobs table"

# Manually create empty migration
alembic revision -m "Add custom index"
```

**Autogenerate Limitations:** Cannot detect column type changes, renames, or constraint modifications. Always review generated migrations.

### Apply Migration

```bash
# Apply all pending migrations
alembic upgrade head

# Rollback one migration
alembic downgrade -1

# Check current version
alembic current
```

### Migration Checklist

Copy this checklist and track progress:
- [ ] Review autogenerated migration for accuracy
- [ ] Add indexes for foreign keys and filtered columns
- [ ] Test migration on dev database
- [ ] Test rollback with `downgrade -1`
- [ ] Backup production database
- [ ] Apply migration to production with `upgrade head`
- [ ] Verify schema with `alembic current`

## Testing Database Operations

### Fixture for Test Database

```python
# toolbox/backend/tests/conftest.py
import pytest
from sqlalchemy.ext.asyncio import create_async_engine, AsyncSession, async_sessionmaker
from app.models.db_models import Base

@pytest.fixture
async def db_session():
    # Use in-memory SQLite for tests (or separate test PostgreSQL)
    engine = create_async_engine("sqlite+aiosqlite:///:memory:")
    
    async with engine.begin() as conn:
        await conn.run_sync(Base.metadata.create_all)
    
    AsyncTestSession = async_sessionmaker(engine, class_=AsyncSession, expire_on_commit=False)
    
    async with AsyncTestSession() as session:
        yield session
    
    await engine.dispose()
```

### Test CRUD Operations

```python
# toolbox/backend/tests/test_db.py
import pytest
from app.models.db_models import Job
from sqlalchemy import select

@pytest.mark.asyncio
async def test_create_job(db_session):
    job = Job(id="test-123", user_email="test@example.com", tool_name="extract")
    db_session.add(job)
    await db_session.commit()
    
    result = await db_session.execute(select(Job).where(Job.id == "test-123"))
    saved_job = result.scalar_one()
    
    assert saved_job.user_email == "test@example.com"
    assert saved_job.status == "pending"
```

### Iterate-Until-Pass Pattern

1. Write test with expected behavior
2. Run tests: `pytest -v tests/test_db.py`
3. If tests fail, fix model/query and repeat step 2
4. Only proceed when all tests pass

**Common Test Failures:**
- Missing `await` on queries → Add `await`
- Lazy load errors → Use `selectinload()` or `expire_on_commit=False`
- Foreign key violations → Check relationship definitions

## Production Deployment

### Docker Compose Setup

```yaml
# toolbox/docker-compose.yml
services:
  db:
    image: postgres:15
    environment:
      POSTGRES_USER: postgres
      POSTGRES_PASSWORD: postgres
      POSTGRES_DB: toolbox
    ports:
      - "5432:5432"
    volumes:
      - postgres_data:/var/lib/postgresql/data

  backend:
    build: .
    environment:
      DATABASE_URL: postgresql+asyncpg://postgres:postgres@db:5432/toolbox
      DATABASE_ENABLED: "true"
    depends_on:
      - db

volumes:
  postgres_data:
```

### Cloud Run Deployment (Google Cloud SQL)

**This project uses Firestore in production.** If enabling PostgreSQL:

1. Create Cloud SQL instance:
   ```bash
   gcloud sql instances create toolbox-db \
     --database-version=POSTGRES_15 \
     --tier=db-f1-micro \
     --region=us-central1
   ```

2. Set connection string:
   ```bash
   gcloud run services update table-rock-tools \
     --set-env-vars DATABASE_URL="postgresql+asyncpg://user:pass@/dbname?host=/cloudsql/PROJECT:REGION:INSTANCE"
   ```

3. Add Cloud SQL proxy:
   ```dockerfile
   # Dockerfile
   RUN wget https://dl.google.com/cloudsql/cloud_sql_proxy.linux.amd64 -O cloud_sql_proxy
   CMD ["./cloud_sql_proxy", "-instances=PROJECT:REGION:INSTANCE=tcp:5432", "&", "uvicorn", "app.main:app"]
   ```

**Cost Warning:** Cloud SQL costs ~$10/month minimum. For this project's scale, **Firestore is more cost-effective**.

## Troubleshooting Connection Issues

### Error: "Connection Refused"

**Symptoms:** `asyncpg.exceptions.ConnectionRefusedError`

**Causes:**
1. PostgreSQL not running: `docker-compose up db`
2. Wrong port: Check `DATABASE_URL` port matches PostgreSQL
3. Firewall blocking connection: Allow port 5432

**Fix:**

```bash
# Check PostgreSQL is running
docker ps | grep postgres

# Test connection directly
psql postgresql://postgres:postgres@localhost:5432/toolbox

# Verify DATABASE_URL format
echo $DATABASE_URL
# Should be: postgresql+asyncpg://user:pass@host:port/dbname
```

### Error: "SSL Connection Required"

**Symptoms:** `asyncpg.exceptions.InvalidAuthorizationSpecificationError`

**Causes:** Cloud SQL requires SSL connections by default

**Fix:**

```python
# Add SSL mode to connection string
DATABASE_URL=postgresql+asyncpg://user:pass@host/db?sslmode=require
```

### Error: "Pool Timeout"

**Symptoms:** Requests hang, then timeout after 30s

**Causes:**
1. Connection pool exhausted (default 5 connections)
2. Sessions not closed (missing context manager)
3. Long-running queries blocking pool

**Fix:**

```python
# Increase pool size
engine = create_async_engine(
    settings.database_url,
    pool_size=20,  # Default: 5
    max_overflow=10,  # Default: 10
    pool_timeout=30,  # Seconds to wait for connection
)

# Always use context managers
async with AsyncSessionLocal() as session:
    # Automatically released on exit
    pass
```

### Debugging Checklist

Copy this checklist when debugging connection issues:
- [ ] PostgreSQL service is running (`docker ps` or `systemctl status postgresql`)
- [ ] `DATABASE_URL` format is correct: `postgresql+asyncpg://user:pass@host:port/dbname`
- [ ] Firewall allows port 5432
- [ ] `DATABASE_ENABLED=true` in environment
- [ ] All queries use `await` (check for missing async)
- [ ] All sessions use context managers (`async with`)
- [ ] Check logs for specific error messages (`docker logs <container>`)
