## Docker Architecture

### Multi-Stage Dockerfile

**Stage 1: Frontend Build (Node 20)**
- Base: `node:20-slim`
- Workdir: `/app/frontend`
- Copy: `package*.json`, install deps, copy `src/`, build to `dist/`
- Output: Static assets in `/app/frontend/dist`

**Stage 2: Python Runtime (Python 3.11)**
- Base: `python:3.11-slim`
- Workdir: `/app`
- Copy: `requirements.txt`, install deps, copy `backend/app/`, copy frontend `dist/`
- Serve: Frontend static files from `/app/dist`, FastAPI on port 8080
- Health check: `curl -f http://localhost:8080/api/health` (30s interval)
- Command: `python3 -m uvicorn app.main:app --host 0.0.0.0 --port $PORT`

**Key Patterns:**
- Use `python3` not `python` (macOS compatibility)
- Port 8080 for Cloud Run (set via $PORT env var)
- Health endpoint required: `/api/health` (FastAPI route in `main.py`)
- Frontend dist served by FastAPI static files middleware

### docker-compose.yml

**Services:**
1. **db** (postgres:15)
   - Port: 5432 (host) → 5432 (container)
   - Env: `POSTGRES_USER=postgres`, `POSTGRES_PASSWORD=postgres`, `POSTGRES_DB=toolbox`
   - Volume: `postgres_data:/var/lib/postgresql/data`
   - Used only when `DATABASE_ENABLED=true` (default: false, use Firestore)

2. **backend** (build: `.`, target: `runtime`)
   - Port: 8000 (host) → 8000 (container)
   - Env: `PORT=8000`, `DATABASE_URL=postgresql+asyncpg://postgres:postgres@db:5432/toolbox`, `ENVIRONMENT=development`
   - Volumes: `./backend/app:/app/app` (hot reload), `./backend/data:/app/data` (persistent storage)
   - Depends on: `db`
   - Command: `python3 -m uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload`

3. **frontend** (node:20-slim)
   - Port: 5173 (host) → 5173 (container)
   - Workdir: `/app/frontend`
   - Volumes: `./frontend:/app/frontend`, `/app/frontend/node_modules` (anonymous volume)
   - Env: `VITE_API_BASE_URL=/api`
   - Command: `npm run dev -- --host 0.0.0.0`

**Volumes:** `postgres_data` (persistent PostgreSQL data)

## GitHub Actions CI/CD

**File:** `.github/workflows/deploy.yml`

**Trigger:** Push to `main` branch

**Workflow Steps:**
1. Checkout code
2. Authenticate to Google Cloud (workload identity or service account key)
3. Set up Cloud SDK
4. Build frontend: `cd toolbox/frontend && npm ci && npm run build`
5. Deploy to Cloud Run: `gcloud run deploy table-rock-tools --source ./toolbox --project tablerockenergy --region us-central1 --allow-unauthenticated --platform managed --memory 1Gi --cpu 1 --timeout 600s --max-instances 10 --min-instances 0`

**Environment Secrets Required:**
- `GCP_PROJECT_ID` (tablerockenergy)
- `GCP_SA_KEY` or workload identity pool
- `GOOGLE_APPLICATION_CREDENTIALS` (service account JSON for Firebase Admin, Firestore, GCS)
- `GCS_BUCKET_NAME` (table-rock-tools-storage)
- `GCS_PROJECT_ID` (tablerockenergy)

**Cloud Run Service Config:**
- Service: `table-rock-tools`
- Region: `us-central1`
- URL: `https://tools.tablerocktx.com`
- Resources: 1 CPU, 1Gi memory, 600s timeout, 0-10 instances
- Port: 8080 (set via $PORT env var)
- Health check: `/api/health` endpoint

## Environment Variables

**Frontend (Vite):**
- `VITE_API_BASE_URL` (default: `/api`) - API proxy base URL

**Backend (FastAPI):**
- `PORT` (default: 8000, production: 8080) - Server port
- `ENVIRONMENT` (default: `development`) - `development` or `production`
- `DEBUG` (default: `false`) - Enable debug logging
- `MAX_UPLOAD_SIZE_MB` (default: 50) - Max file upload size
- `DATABASE_URL` (default: `postgresql+asyncpg://postgres:postgres@localhost:5432/toolbox`) - PostgreSQL connection
- `DATABASE_ENABLED` (default: `false`) - Enable PostgreSQL (use Firestore by default)
- `FIRESTORE_ENABLED` (default: `true`) - Enable Firestore persistence
- `GCS_BUCKET_NAME` (default: `table-rock-tools-storage`) - GCS bucket name
- `GCS_PROJECT_ID` (default: `tablerockenergy`) - GCP project ID
- `GOOGLE_APPLICATION_CREDENTIALS` - Path to GCP service account key JSON

**No `.env.example` file exists.** Environment variables are documented in `backend/app/core/config.py` via Pydantic Settings with `@property` helpers.

## Key Patterns from This Codebase

1. **Python Command:** Always use `python3`, never `python` (macOS compatibility)
2. **Port Configuration:** Port 8000 for local dev, 8080 for Cloud Run (via $PORT env var)
3. **Storage Fallback:** GCS primary, local `backend/data/` fallback (transparent via `StorageService`)
4. **Database Priority:** Firestore is primary DB, PostgreSQL optional (disabled by default)
5. **Health Check Required:** FastAPI must expose `/api/health` endpoint for Cloud Run health checks
6. **Frontend Proxy:** Vite dev server proxies `/api` to `http://localhost:8000` (no CORS issues)
7. **Multi-Stage Build:** Node 20 build → Python 3.11 runtime (optimized image size)
8. **Allowlist Auth:** Firebase token verification + JSON allowlist at `backend/data/allowed_users.json`
9. **RRC SSL Adapter:** RRC website requires custom SSL adapter (`RRCSSLAdapter` in `rrc_data_service.py`)
10. **Firestore Batching:** Batch operations commit every 500 documents (Firestore limit)

## CRITICAL for This Project

### Dockerfile Rules
- **Node stage:** Must build frontend to `dist/` before Python stage
- **Python stage:** Must copy frontend `dist/` to `/app/dist` for FastAPI static file serving
- **Port exposure:** Expose port 8080 (Cloud Run default)
- **Health check:** Must include `curl -f http://localhost:8080/api/health` with 30s interval
- **Command:** Use `python3 -m uvicorn app.main:app --host 0.0.0.0 --port $PORT` (respect $PORT env var)
- **Working directory:** Backend must run from `/app` (not `/app/backend`)

### docker-compose Rules
- **Port mapping:** Backend 8000:8000, Frontend 5173:5173, PostgreSQL 5432:5432
- **Hot reload:** Mount `./backend/app` and `./frontend` as volumes for dev
- **PostgreSQL:** Optional, only used when `DATABASE_ENABLED=true`
- **Data persistence:** Mount `./backend/data` for local storage (uploads, RRC CSVs, allowlist)
- **Environment:** Set `ENVIRONMENT=development` for backend
- **Frontend command:** `npm run dev -- --host 0.0.0.0` (allow external connections)

### GitHub Actions Rules
- **Trigger:** Only on push to `main` branch
- **Build order:** Build frontend FIRST, then deploy (Cloud Run builds backend)
- **Project:** Must use `tablerockenergy` project
- **Region:** Must use `us-central1` region
- **Service name:** Must use `table-rock-tools` (do not rename)
- **Authentication:** Must authenticate to GCP before deploy (workload identity or service account key)
- **Secrets:** Never commit `GOOGLE_APPLICATION_CREDENTIALS` or `GCP_SA_KEY` to repo
- **Source deployment:** Use `--source ./toolbox` (Cloud Run builds from source, not pre-built image)

### Cloud Run Configuration
- **Port:** Must listen on port 8080 (set via $PORT env var)
- **Health endpoint:** Must respond to GET `/api/health` (200 OK)
- **Timeout:** Set to 600s for long-running file processing operations
- **Memory:** 1Gi (minimum for Pandas/PDF processing)
- **CPU:** 1 vCPU
- **Scaling:** 0-10 instances (scale to zero when idle)
- **Allow unauthenticated:** Yes (Firebase Auth handles authorization at application level)
- **Environment variables:** Set `FIRESTORE_ENABLED=true`, `DATABASE_ENABLED=false`, `GCS_BUCKET_NAME`, `GCS_PROJECT_ID`, `GOOGLE_APPLICATION_CREDENTIALS` (mounted as secret)

### Security Rules
- **Never commit:** `GOOGLE_APPLICATION_CREDENTIALS`, service account keys, Firebase config (use env vars)
- **Allowlist:** Default admin is `james@tablerocktx.com`, stored in `backend/data/allowed_users.json`
- **Secrets management:** Use GitHub Secrets for CI/CD, Google Secret Manager for Cloud Run
- **Docker images:** Scan for vulnerabilities before deploy (use `docker scan` or Trivy)
- **Multi-stage builds:** Minimize attack surface by excluding dev dependencies from runtime image

### Troubleshooting Common Issues

1. **Build failures:** Check Node 20+ and Python 3.11+ versions, verify `requirements.txt` and `package.json` integrity
2. **Port conflicts:** Ensure ports 5173, 8000, 5432 are available on host
3. **GCS unavailable:** Storage service falls back to local `backend/data/` automatically (check logs)
4. **Health check failures:** Verify `/api/health` endpoint returns 200 OK, check backend logs
5. **Frontend 404 errors:** Ensure FastAPI static files middleware serves from correct `dist/` path
6. **PostgreSQL connection errors:** Check `DATABASE_ENABLED=false` (use Firestore by default)
7. **RRC download errors:** Custom SSL adapter required (`RRCSSLAdapter` in `rrc_data_service.py`)
8. **Cloud Run deployment hangs:** Check build logs, verify frontend builds successfully, check health endpoint
