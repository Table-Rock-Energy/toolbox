**Interfaces (define in this file):**
```typescript
interface BatchProgress {
  currentBatch: number
  totalBatches: number
  failedBatches: number
  skippedEntries: number
  currentStep: PipelineStep
  batchTimings: number[]  // ms per completed batch, for ETA
}

interface OperationState {
  tool: string
  status: PipelineStatus           // 'idle' | 'running' | 'completed' | 'error'
  stepStatuses: StepStatus[]
  batchProgress: BatchProgress | null
  enrichmentChanges: Map<string, EnrichmentCellChange>
  completedSteps: Set<PipelineStep>
  entriesSnapshot: Record<string, unknown>[] | null  // Pre-pipeline for undo
  resultEntries: Record<string, unknown>[] | null     // Final results for auto-restore
  errorMessage: string | null
}

interface StartOperationOpts {
  tool: string
  entries: Record<string, unknown>[]
  updateEntries: (entries: Record<string, unknown>[]) => void
  editedFields: Map<string, unknown>
  keyField: string
  featureFlags: { cleanUpEnabled: boolean; validateEnabled: boolean; enrichEnabled: boolean }
  sourceData?: Record<string, unknown>[]
}

interface OperationActions {
  startOperation: (opts: StartOperationOpts) => void
  abortOperation: () => void
  undoOperation: () => void
  clearOperation: () => void
  getResultsForTool: (tool: string) => Record<string, unknown>[] | null
}
```

**Split context pattern (to avoid re-render storms per RESEARCH.md Pitfall 3):**
- `OperationStateContext = createContext<OperationState | null>(null)` — volatile, changes per batch
- `OperationActionsContext = createContext<OperationActions | null>(null)` — stable functions

**OperationProvider component:**
1. State: `useState<OperationState | null>(null)`, `useState<boolean>(false)` for cancel confirmation pending
2. `abortControllerRef = useRef<AbortController | null>(null)`
3. `updateEntriesRef = useRef<((entries: Record<string, unknown>[]) => void) | null>(null)` — stores the active tool page's updateEntries callback

4. **beforeunload handler** (useEffect):
   ```typescript
   useEffect(() => {
     const handleBeforeUnload = () => { abortControllerRef.current?.abort() }
     window.addEventListener('beforeunload', handleBeforeUnload)
     return () => window.removeEventListener('beforeunload', handleBeforeUnload)
   }, [])
   ```

5. **startOperation function:**
   - Check if `operation?.status === 'running'`. If yes, set `pendingOperation` state and show cancel confirmation (do NOT start yet). If no, proceed directly.
   - Create AbortController, store in ref
   - Set operation state to `{ tool, status: 'running', stepStatuses: [...], batchProgress: null, ... }`
   - Store `updateEntries` in ref (for progressive apply)
   - Snapshot entries in `entriesSnapshot`
   - Call `runPipeline(...)` async function

6. **runPipeline async function** (core engine):
   - Determine enabled steps from featureFlags
   - Initialize stepStatuses as pending
   - For each step:
     a. Set step to 'active'
     b. Slice entries into batches of BATCH_SIZE (25)
     c. `const totalBatches = Math.ceil(entries.length / BATCH_SIZE)`
     d. Loop over batches:
        - Check `signal.aborted`, break if true
        - `const batchStart = i * BATCH_SIZE`
        - `const batch = currentEntries.slice(batchStart, batchStart + BATCH_SIZE)`
        - `const batchStartTime = Date.now()`
        - try/catch the API call: `await apiMethod(tool, batch, undefined, sourceData)`
        - On success:
          - `const elapsed = Date.now() - batchStartTime`
          - Push elapsed to `batchTimings`
          - **Apply results with index offset**: `const globalIndex = batchStart + change.entry_index` (CRITICAL: backend returns indices relative to the batch slice, not the full array)
          - For each change: `currentEntries[globalIndex][change.field] = change.proposed_value`
          - Track in allChanges map keyed by `${globalIndex}:${change.field}`
          - Progressive apply: `updateEntriesRef.current?.(currentEntries.map(e => ({...e})))`
          - Update batchProgress state: `{ currentBatch: i + 1, totalBatches, failedBatches, skippedEntries, currentStep: step, batchTimings }`
        - On error (catch):
          - `failedBatches++`
          - `skippedEntries += batch.length`
          - Update batchProgress with failure counts
          - `continue` (RESIL-03: skip and continue)
     e. After all batches for this step: mark step completed with changesApplied count
     f. Reset batchProgress to null before moving to next step
   - After all steps: set `status: 'completed'`, store `resultEntries`, clear abortControllerRef

   **CRITICAL — use local variables for the batch loop** (not React state, per RESEARCH.md Pitfall 1):
   - `let currentEntries = entries.map(e => ({...e}))` — mutated in-place across batches
   - `const allChanges = new Map<string, EnrichmentCellChange>()` — accumulated across steps
   - `const batchTimings: number[]` — reset per step
   - Only push to React state for rendering (setOperation with new batchProgress)

7. **abortOperation:** `abortControllerRef.current?.abort()`, set status to 'idle', clear abortControllerRef

8. **undoOperation:** If entriesSnapshot exists, call `updateEntriesRef.current?.(entriesSnapshot)`, reset operation to null

9. **clearOperation:** Set operation to null

10. **getResultsForTool:** Return `operation?.tool === tool && operation?.status === 'completed' ? operation.resultEntries : null`

11. **Memoize actions** with `useMemo` so OperationActionsContext value is stable (functions wrapped in useCallback)

12. **Render:**
    ```tsx
    <OperationStateContext.Provider value={operation}>
      <OperationActionsContext.Provider value={actions}>
        {children}
      </OperationActionsContext.Provider>
    </OperationStateContext.Provider>
    ```

**Export hooks:**
```typescript
export function useOperationState() {
  return useContext(OperationStateContext)
}
export function useOperationActions() {
  const ctx = useContext(OperationActionsContext)
  if (!ctx) throw new Error('useOperationActions must be used within OperationProvider')
  return ctx
}
// Convenience: combines both
export function useOperationContext() {
  return { operation: useOperationState(), ...useOperationActions() }
}
```
