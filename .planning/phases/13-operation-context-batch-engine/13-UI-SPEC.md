## Component Inventory

New and modified components for this phase:

### New: Cancel Confirmation Dialog

Shown when user starts a new enrichment operation while one is already running (one-at-a-time constraint).

- **Trigger:** `startOperation()` called while `operation.status === 'running'`
- **Layout:** Same modal pattern as EnrichmentModal (centered, backdrop blur, rounded-xl, shadow-2xl, max-w-md)
- **Buttons:** "Keep Running" (secondary, left) and "Cancel & Start New" (destructive, right)
- **Button styles:**
  - "Keep Running": `px-4 py-2 border border-gray-300 text-gray-700 rounded-lg hover:bg-gray-50`
  - "Cancel & Start New": `px-4 py-2 bg-red-500 text-white rounded-lg hover:bg-red-600`

### Modified: EnrichmentModal

Extended with batch-level progress sub-section.

- **Batch progress row:** Appears below the overall progress bar when a step is actively processing batches. Uses `text-xs text-gray-500` for the batch label. Format: "Clean Up: Batch 3 of 8 -- ~45s remaining"
- **Batch sub-progress bar:** Nested inside the active step row. Same styling as overall progress bar (`h-2 bg-gray-200 rounded-full` track, `bg-tre-teal rounded-full` fill) but scoped to batch completion within the current step.
- **Failure summary:** When `failedBatches > 0` at step completion, show inline below the step label: `text-xs text-amber-600` with "{skipped} entries skipped ({failed} batch failed)". Uses amber (not red) to distinguish partial failure from total step failure.
- **Amber addition:** `#d97706` amber-600 for partial failure text only. This is the single new color introduced.

### Unchanged (context-driven props only)

Tool pages (Extract, Title, Proration, Revenue) change from direct `useEnrichmentPipeline` to `useOperationContext()` consumers. No visual changes to these pages. EnrichmentModal continues to render inside tool pages, deriving `isOpen` from context state.

## Interaction Contracts

### Navigate Away During Operation
- **Behavior:** Modal closes. Operation continues silently in OperationContext. No toast, no warning, no confirmation.
- **Return:** If operation still running, modal auto-reopens showing current progress. If completed, results auto-restore to preview table without modal.

### Cancel Confirmation Flow
1. User clicks "Run Enrichment" on Tool B while Tool A operation is running
2. Cancel confirmation dialog appears (see Component Inventory above)
3. "Keep Running" dismisses dialog, no state change
4. "Cancel & Start New" aborts current operation, clears state, starts new operation on Tool B

### Tab Close / Browser Close
- `beforeunload` fires, AbortController aborts all in-flight fetch requests
- No confirmation dialog (browser handles its own beforeunload if needed)

### Batch Failure
- Failed batch: step continues with remaining batches
- Failed batch entries remain unchanged in preview (no error markers on individual rows)
- Modal shows inline failure count in amber text below the step

### Progressive Auto-Apply
- Each batch's results apply immediately to the preview table as they complete
- User sees rows updating live if they stay on the tool page
- No visual indicator per-row for "just updated" (no flash/highlight) -- changes are silent
