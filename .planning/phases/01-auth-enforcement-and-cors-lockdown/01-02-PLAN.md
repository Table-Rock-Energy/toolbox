**api.ts** — Add 401 interceptor to ApiClient:
- Add private field: `private onUnauthorized: (() => Promise<boolean>) | null = null`
- Add private field: `private isRefreshing = false`
- Add public method: `setUnauthorizedHandler(handler: () => Promise<boolean>): void`
- In the `request()` method, after `if (!response.ok)` block (line 69), add 401 interception BEFORE the existing error handling:
  - If `response.status === 401` AND `this.onUnauthorized` AND `!this.isRefreshing`:
    - Set `this.isRefreshing = true`
    - Call `await this.onUnauthorized()` — if it returns true (token refreshed), set `isRefreshing = false` and retry `return this.request<T>(endpoint, options)`
    - In finally block, set `this.isRefreshing = false`
    - If refresh returned false, fall through to existing error handling
- Apply the same 401 interception pattern in `uploadFile()` method

**AuthContext.tsx** — Three changes:
1. **Fail-closed**: In `checkAuthorization` catch block (line 53-56), change `return true` to `return false`. This is the core fix — deny access when backend is unreachable.

2. **Backend reachability detection**: Add a new state `backendReachable` (default `true`). Before calling `checkAuthorization` in the `onAuthStateChanged` callback, probe `/api/health` with a fetch. If it fails (network error), set `backendReachable = false` and skip authorization check (user stays unauthorized due to fail-closed). If it succeeds, set `backendReachable = true` and proceed with authorization check. Export `backendReachable` in the context value.

3. **Token refresh callback**: After `onAuthStateChanged` sets up the user and calls `api.setAuthToken`, also call `api.setUnauthorizedHandler` with an async callback that:
   - Gets the current Firebase user
   - Calls `user.getIdToken(true)` to force refresh
   - If successful, calls `api.setAuthToken(newToken)` and returns `true`
   - If refresh fails, calls `signOut()` (clear auth state), sets `authError` to "Your session has expired. Please sign in again.", and returns `false`

4. **Add `backendReachable` to AuthContextType interface** and to the context value object.

**useSSEProgress.ts** — Accept auth token and pass as query parameter:
- Change the hook signature to accept a second parameter: `export function useSSEProgress(jobId: string | null, authToken?: string | null): UseSSEProgressReturn`
- In `connectEventSource` (line 68-70), build URL with token:
  ```typescript
  let url = `/api/ghl/send/${jobId}/progress`
  if (authToken) {
    url += `?token=${encodeURIComponent(authToken)}`
  }
  ```
- The calling component (GhlSendModal or wherever useSSEProgress is used) will need to pass the auth token from `useAuth().getIdToken()`. Check where `useSSEProgress` is called and update the call site to pass the token.
