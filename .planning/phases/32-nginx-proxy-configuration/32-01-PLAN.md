<task>
  <name>Task 1: Add /api/pipeline/ location block for long-running AI inference</name>
  <files>nginx/default.conf</files>
  <read_first>
    - nginx/default.conf
  </read_first>
  <action>
Insert a new location block for `/api/pipeline/` above the default `location /` block, immediately after the existing `/api/ghl/send/` block. The block must:

- `proxy_pass http://127.0.0.1:8080;`
- `proxy_http_version 1.1;`
- Set the standard Host / X-Real-IP / X-Forwarded-For / X-Forwarded-Proto headers (match the existing blocks verbatim)
- `proxy_read_timeout 600s;`
- `proxy_connect_timeout 600s;`
- `proxy_send_timeout 600s;`
- Include a brief comment header, e.g. `# -- Pipeline endpoints (long-running AI inference via Ollama) --`

Do NOT set `proxy_buffering off` for this block — `/api/pipeline/*` returns a single JSON response, not a stream. Buffering on is fine and actually preferred (lets nginx handle slow clients efficiently).

Leave the existing `/api/proration/`, `/api/ghl/send/`, and default `/` blocks unchanged.
  </action>
  <verify>
    <automated>

      grep -A 20 "location /api/pipeline/" nginx/default.conf | grep -q "proxy_read_timeout 600s" &amp;&amp; \
      echo PASS || echo FAIL
    </automated>
  </verify>
  <acceptance_criteria>
    - `grep -c "location /api/pipeline/" nginx/default.conf` returns 1
    - The new block sits between the `/api/ghl/send/` block and the default `location /` block

    - No change to any other existing location block (verify with `git diff nginx/default.conf` — only additions, no unrelated modifications)
  </acceptance_criteria>
