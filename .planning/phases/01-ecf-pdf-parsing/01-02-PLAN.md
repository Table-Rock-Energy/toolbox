**2. Update `backend/app/services/extract/export_service.py` -- filter address-less entries:**

In `_entries_to_dataframe()`, add filtering logic: when building the DataFrame, exclude entries where `section_type` is `"address_unknown"` or `"curative_unknown"` AND the entry has no mailing_address and no city. This implements the user decision that "exports exclude entries without addresses."

The filtering should be:
```python
# Filter address-less entries from exports (ECF section types)
if any(e.section_type for e in entries):
    entries = [
        e for e in entries
        if e.mailing_address or e.city
        or e.section_type not in ("address_unknown", "curative_unknown")
    ]
```

This preserves backward compatibility: non-ECF entries (section_type=None) are never filtered. ECF entries with addresses are kept. Only address-unknown ECF entries without addresses are excluded.
