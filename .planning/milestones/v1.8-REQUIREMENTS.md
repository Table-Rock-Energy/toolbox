## v1.8 Requirements

Requirements for Preview System Overhaul. All 10 requirements shipped.

### Preview Highlights

- [x] **PREV-01**: Enrichment highlights use stable entry keys (_uid/entry_number) instead of array indices, so highlights align with correct rows regardless of filter/sort state
- [x] **PREV-02**: User can click a green (changed) cell to reveal the original value underneath, confirming the change is on the correct row
- [x] **PREV-03**: Rows that were sent to enrichment but had no changes show a subtle checkmark indicator so user knows they were processed
- [x] **PREV-04**: Rows with enrichment changes display blue-tinted background; unenriched rows remain white (preserve existing behavior, fix alignment)

### Filtering

- [x] **FILT-01**: Entity type filters (Individual, Trust, LLC, etc.) correctly exclude non-matching types on Extract, Title, and all tool pages
- [x] **FILT-02**: Filters can be applied/changed at any time (before, during, after enrichment) without breaking highlight alignment or data integrity
- [x] **FILT-03**: Enrichment processes only the currently filtered/visible rows, not the full dataset
- [x] **FILT-04**: Export includes only visible (filtered) rows with all enrichment changes correctly applied

### Proration

- [x] **PROR-01**: RRC lookup tries lease number only first; falls back to district+lease prefix for edge cases; persists new results to Firestore
- [x] **PROR-02**: Fetch-missing progress modal includes a stop button that finishes the current RRC lookup, then stops and returns all results found so far
