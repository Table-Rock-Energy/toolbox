    Add a `prewarm_rrc_cache` async function to `rrc_cache.py` that:
    1. Imports `rrc_data_service` from `rrc_data_service.py`
    2. Calls `await asyncio.to_thread(rrc_data_service._load_lookup)` to load the DataFrame (CPU-bound, must not block event loop per research pitfall 1)
    3. Logs the number of entries loaded: `logger.info("RRC DataFrame pre-warmed: %d entries", len(lookup))`
    4. Does NOT populate the Firestore-backed cache at startup (per research anti-pattern: "Pre-loading all Firestore RRC docs at startup" is too slow with 100K+ docs)

    In `backend/app/main.py` `startup_event()`, add AFTER the existing init blocks (after database init, before the RRC comment):
    ```python
    # Pre-warm RRC DataFrame cache (PERF-02)
    try:
        from app.services.proration.rrc_cache import prewarm_rrc_cache
        await prewarm_rrc_cache()
    except Exception as e:
        logger.warning(f"RRC pre-warm failed (will load on first request): {e}")
    ```

    This must be wrapped in try/except so a pre-warm failure doesn't prevent app startup.
