  <action>
Create `toolbox/frontend/src/pages/GhlPrep.tsx` following the Title.tsx page pattern but significantly simpler (no job history, no filters, no edit modal, no enrichment). The GHL Prep tool is a single-pass transform: upload -> preview -> export.

Structure:
1. **Header**: Icon (use `Repeat` or `RefreshCw` from lucide-react — represents transformation), title "GHL Prep", subtitle "Transform Mineral export CSVs for GoHighLevel import"

2. **State**:
   - `isProcessing` (boolean) — upload in progress
   - `error` (string | null) — error message
   - `result` (TransformResult | null) — the API response result
   - `sortColumn` (string | null) — current sort column
   - `sortDirection` ('asc' | 'desc') — sort direction

3. **Upload Section**: Use `FileUpload` component with `accept=".csv"`, label="Upload Mineral Export", description="Drop your Mineral CSV export here". Show processing spinner when uploading.

4. **Results Section** (shown when result exists):
   a. **Stats Bar**: 4 stat cards in a grid:
      - Total Rows (result.total_count)
      - Names Title-Cased (result.transformed_fields.title_cased)
      - Campaigns Extracted (result.transformed_fields.campaigns_extracted)
      - Phones Mapped (result.transformed_fields.phone_mapped)

   b. **Warnings**: If result.warnings has entries, show yellow warning box with each warning.

   c. **Preview Table**:
      - Scrollable table showing all columns from result.rows
      - Column headers derived from first row's keys (dynamic — not hardcoded since Mineral exports vary)
      - Clickable headers for sorting
      - Paginate: show 50 rows at a time with "Show More" button or simple pagination
      - Max height with vertical scroll
      - Cells truncated with title tooltip for long values

   d. **Export Button**: Fixed at top-right of results section. "Download CSV" button with Download icon. On click:
      - POST to `/api/ghl-prep/export/csv` with `{ rows: result.rows, filename: source_filename }`
      - Receive blob, create download link, click programmatically
      - Filename: `{source_name}_ghl_prep.csv`

5. **Upload Handler** (`handleFilesSelected`):
   - Create FormData with file
   - POST to `/api/ghl-prep/upload` with X-User-Email and X-User-Name headers
   - Parse response as UploadResponse
   - Set result state

6. **Sorting**: Client-side sort on the rows array. When user clicks column header, toggle sort. Use `useMemo` for sorted rows.

7. **Reset**: "Upload New File" button to clear result and show upload area again.

Use Tailwind classes matching existing tool pages:
- `bg-white rounded-xl border border-gray-200` for cards
- `font-oswald` for headers
- `text-tre-navy` for primary text
- `bg-tre-teal` / `text-tre-teal` for accent elements
- Stats use `text-2xl font-oswald font-semibold` pattern from Title.tsx
