```python
class BulkContactData(BaseModel):
    """Single contact in a bulk send request. All field names match GHL field names."""
    mineral_contact_system_id: str = Field(..., description="Stable source identifier for result tracking")
    first_name: Optional[str] = None
    last_name: Optional[str] = None
    email: Optional[str] = None
    phone: Optional[str] = None
    address1: Optional[str] = None
    city: Optional[str] = None
    state: Optional[str] = None
    postal_code: Optional[str] = None

class BulkSendRequest(BaseModel):
    """Request model for bulk contact send."""
    connection_id: str = Field(..., description="GHL connection ID")
    contacts: list[BulkContactData] = Field(..., min_length=1, description="Contacts to send")
    campaign_tag: str = Field(..., min_length=1, description="Campaign tag to apply to all contacts")
    manual_sms: bool = Field(False, description="Apply 'manual sms' tag to all contacts")
    assigned_to: Optional[str] = Field(None, description="GHL user ID for contact owner")
    smart_list_name: Optional[str] = Field(None, description="SmartList/campaign name for reference")

class ContactResult(BaseModel):
    """Per-contact result in bulk send response."""
    mineral_contact_system_id: str
    status: str = Field(description="created | updated | failed | skipped")
    ghl_contact_id: Optional[str] = None
    error: Optional[str] = None

class BulkSendValidationResponse(BaseModel):
    """Response from batch validation (before sending)."""
    valid_count: int
    invalid_count: int
    invalid_contacts: list[ContactResult] = Field(default_factory=list, description="Contacts that failed validation with error details")

class BulkSendResponse(BaseModel):
    """Response from bulk send operation."""
    job_id: str
    total_count: int
    created_count: int
    updated_count: int
    failed_count: int
    skipped_count: int
    results: list[ContactResult]
```

**2. Create `services/ghl/bulk_send_service.py`** with three functions:

**`validate_batch(contacts: list[dict]) -> tuple[list[dict], list[ContactResult]]`:**
- Import `normalize_contact`, `validate_contact` from `normalization.py`
- Iterate all contacts
- For each: check `mineral_contact_system_id` exists (skip with error if missing)
- Normalize contact via `normalize_contact()`
- Validate via `validate_contact()` — checks email format or phone E.164 normalizable
- Return tuple: (valid_contacts_list, invalid_contact_results_list)
- Valid contacts include normalized data + `mineral_contact_system_id` preserved
- Invalid contacts get `ContactResult(status="skipped", error=validation_error_message)`

**`process_batch(connection_id: str, contacts: list[dict], tags: list[str], assigned_to: str | None = None) -> dict`:**
- Lazy import `get_connection` from `connection_service`, `GHLClient` and exceptions from `client`
- Fetch connection with `decrypt_token=True`, raise ValueError if not found
- Build tags list (always includes campaign_tag, optionally "manual sms")
- Create ONE `GHLClient` instance outside the loop (critical: shared rate limiter)
- Loop through contacts sequentially with `async for`
- For each contact: build contact_data dict with tags + assigned_to, call `client.upsert_contact()`
- On success: append `ContactResult(status=result["action"], ghl_contact_id=result["ghl_contact_id"])`
- On `GHLAPIError` or any exception: append `ContactResult(status="failed", error=str(e))`, continue processing (skip-and-continue)
- After loop: check if any contacts have multiple campaign tags by examining existing tags in the upsert response — if contact was "updated" and already had a different campaign-style tag, add a note via client (discretion: format as "Note: Contact appears in multiple campaigns: {tag1}, {tag2}")
- Return dict with `created_count`, `updated_count`, `failed_count`, `total_count`, `results`
- Use `logger = logging.getLogger(__name__)` for logging failures

**`persist_send_job(job_id: str, connection_id: str, campaign_name: str, result_data: dict, user_id: str | None = None) -> None`:**
- Lazy import `get_firestore_client` from `firestore_service`
- Build Firestore document with: job_id, tool="ghl_send", connection_id, campaign_name, created_count, updated_count, failed_count, skipped_count, total_count, results (full list), user_id, created_at (UTC), status="completed"
- Write to `jobs` collection with `job_id` as document ID
- Wrap in try/except — log warning on Firestore failure but don't raise (non-critical)

Use `from __future__ import annotations` at top. Use lazy imports for Firestore and GHL client inside functions.
