    mineral_contact_system_id: str

    error: Optional[str] = None

class BulkSendValidationResponse(BaseModel):
    valid_count: int
    invalid_count: int
    invalid_contacts: list[ContactResult]

class BulkSendResponse(BaseModel):
    job_id: str
    total_count: int
    created_count: int
    updated_count: int
    failed_count: int
    skipped_count: int
    results: list[ContactResult]

**Multi-step flow:**

*Step 1 — Idle (form visible):* User fills form (connection, tag, owner, SmartList, SMS checkbox). "Validate & Send" button is primary action.

*Step 2 — Validating:* User clicks "Validate & Send". Show loading spinner. Build `BulkSendRequest` from form state:
- Map `rows` to `BulkContactData[]` — for each row, extract fields matching BulkContactData keys. The `mineral_contact_system_id` comes from the row's "Mineral Contact System Id" column (or similar key — check for case variations: `mineral_contact_system_id`, `Mineral Contact System Id`).
- Call `ghlApi.validateBatch(request)`
- On success: set `validationResult`, move to 'confirmed' step
- On error: set `sendError`, stay on 'idle'

*Step 3 — Confirmed (validation split):* Show validation results:
- Green text: "{validCount} contacts ready to send"
- If invalidCount > 0: amber warning "{invalidCount} contacts will be skipped (missing email/phone)"
- Two buttons: "Cancel" (go back to idle) and "Send {validCount} Contacts" (primary teal button)

*Step 4 — Sending:* User clicks "Send X Contacts". Show loading spinner with "Sending contacts to GHL..." message. Call `ghlApi.bulkSend(request)` with same request data.
- On success: set `sendResult`, move to 'results' step
- On error: set `sendError`, show error message with "Try Again" button

*Step 5 — Results:* Show result summary:
- Green: "{createdCount} created"
- Blue: "{updatedCount} updated"
- Red (if any): "{failedCount} failed"
- Amber (if any): "{skippedCount} skipped (validation)"
- If failed > 0: expandable section showing failed contact IDs and error messages
- Single "Close" button
