Create `toolbox/backend/app/services/ghl/connection_service.py` with Firestore CRUD operations for GHL connections.

**Module structure:**
- `from __future__ import annotations`
- `logger = logging.getLogger(__name__)`
- `GHL_CONNECTIONS_COLLECTION = "ghl_connections"`

**Functions:**

1. `async def create_connection(name, token, location_id, notes=None) -> dict`:
   - Encrypt token with `encrypt_value(token)`.
   - Store `token_last4 = token[-4:] if len(token) >= 4 else ""`.
   - Create Firestore document with fields: name, encrypted_token, token_last4, location_id, notes, validation_status="pending", created_at, updated_at.
   - Use `db.collection(GHL_CONNECTIONS_COLLECTION).document()` for auto-generated ID.
   - Return dict with `id` and all fields (minus encrypted_token, plus token_last4).

2. `async def get_connection(connection_id, decrypt_token=False) -> Optional[dict]`:
   - Fetch from Firestore by doc ID.
   - If `decrypt_token=True`, decrypt the encrypted_token field and add as `token`.
   - ALWAYS remove `encrypted_token` from returned dict.
   - Add `id` from doc.id.
   - Return None if not found.

3. `async def list_connections() -> list[dict]`:
   - Query all docs from GHL_CONNECTIONS_COLLECTION.
   - For each: remove encrypted_token, add id, return list sorted by name.

4. `async def update_connection(connection_id, name=None, token=None, location_id=None, notes=None) -> Optional[dict]`:
   - Fetch existing connection (with decrypt if token is not being changed).
   - Build update_data dict with only non-None fields.
   - If `token` provided: encrypt new token, update token_last4, set validation_status="pending".
   - Always set updated_at.
   - Update Firestore doc. Return updated connection (via get_connection).

5. `async def delete_connection(connection_id) -> bool`:
   - Check doc exists first. Delete if found. Return True/False.

6. `async def validate_connection(connection_id) -> dict`:
   - Fetch connection with decrypted token.
   - If not found, raise ValueError.
   - Create GHLClient with decrypted token and location_id.
   - Call `client.get_users()` to validate.
   - On success: update validation_status to "valid" in Firestore. Parse users from response.
   - On GHLAuthError: update validation_status to "invalid". Return error details.
   - On other errors: update validation_status to "invalid". Return error details.
   - Return dict with: valid (bool), error (str or None), users (list of user dicts).

7. `async def get_connection_users(connection_id) -> list[dict]`:
   - Fetch connection with decrypted token.
   - Create GHLClient, call get_users().
   - Parse and return list of user dicts (id, name, email, role).

8. `async def upsert_contact_via_connection(connection_id, contact_data: dict) -> dict`:
   - Fetch connection with decrypted token.
   - If not found, raise ValueError.
   - Create GHLClient with decrypted token and location_id.
   - Call `client.upsert_contact(contact_data)`.
   - Return result dict with: success, action, ghl_contact_id, error.

**Key rules:**
- NEVER return encrypted_token to callers. Always pop it from returned dicts.
- Log operations at INFO level without PII: "Created connection {id}", "Deleted connection {id}".
- Use lazy imports for firestore_service and encryption inside functions (follow project pattern).
- All functions are `async def`.

1. Create `toolbox/backend/app/api/ghl.py` with FastAPI router:

**Router setup:**
```python
from fastapi import APIRouter, Depends, HTTPException
from app.core.auth import require_auth
from app.models.ghl import (
    GHLConnectionCreate, GHLConnectionUpdate, GHLConnectionResponse,
    ContactUpsertRequest, ContactUpsertResponse, GHLValidationResult,
    GHLUserResponse,
)

router = APIRouter()
```

**Endpoints (all require `user: dict = Depends(require_auth)`):**

a. `GET /connections` — List all GHL connections.
   - Call `list_connections()`.
   - Return `{"connections": [GHLConnectionResponse(...) for each]}`.

b. `POST /connections` — Create and validate a new connection.
   - Accept `GHLConnectionCreate` body.
   - Call `create_connection(name, token, location_id, notes)`.
   - Then call `validate_connection(connection_id)` to validate immediately.
   - Return `{"connection": GHLConnectionResponse(...), "validation": GHLValidationResult(...)}`.
   - Wrap in try/except: on any error creating, raise HTTPException(400).

c. `PUT /connections/{connection_id}` — Update existing connection.
   - Accept `GHLConnectionUpdate` body.
   - Call `update_connection(connection_id, ...)`.
   - If None returned (not found), raise HTTPException(404).
   - If token was changed (data.token is not None), re-validate.
   - Return `{"connection": GHLConnectionResponse(...)}`.

d. `DELETE /connections/{connection_id}` — Delete connection.
   - Call `delete_connection(connection_id)`.
   - If False (not found), raise HTTPException(404).
   - Return `{"deleted": True}`.

e. `POST /connections/{connection_id}/validate` — Re-validate existing connection.
   - Call `validate_connection(connection_id)`.
   - On ValueError (not found): HTTPException(404).
   - Return `GHLValidationResult(...)`.

f. `GET /connections/{connection_id}/users` — Fetch GHL users for contact owner dropdown.
   - Call `get_connection_users(connection_id)`.
   - On ValueError (not found): HTTPException(404).
   - Return `{"users": [GHLUserResponse(...) for each]}`.

g. `POST /contacts/upsert` — Upsert a single contact to GHL.
   - Accept `ContactUpsertRequest` body.
   - Call `upsert_contact_via_connection(connection_id, contact_dict)`.
   - On ValueError: HTTPException(400) with error detail.
   - On GHLAPIError: HTTPException(502) with GHL error detail (pass through per user decision).
   - Return `ContactUpsertResponse(...)`.

**Error handling pattern:**
- Use lazy imports inside each handler: `from app.services.ghl.connection_service import ...`
- Catch `GHLAPIError`, `GHLAuthError` from `app.services.ghl.client` — map to appropriate HTTP status codes (401 for auth, 502 for API errors, 429 for rate limit).
- Use `logger = logging.getLogger(__name__)` for error logging.

   - Add: `app.include_router(ghl_router, prefix="/api/ghl", tags=["ghl"])`
   - Place after the existing `ghl_prep_router` line for logical grouping.
