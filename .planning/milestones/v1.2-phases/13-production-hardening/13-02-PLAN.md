5. Add SSE reconnection with exponential backoff. Currently the hook creates an EventSource and relies on browser default reconnect. Enhance it:
- Add a `reconnectAttempt` ref (starts at 0)
- On `onerror`, if `eventSource.readyState === EventSource.CLOSED`:
  - If `reconnectAttempt.current < 5`:
    - Calculate backoff: `Math.min(1000 * 2 ** reconnectAttempt.current, 30000)` (1s, 2s, 4s, 8s, 16s, capped at 30s)
    - Set a timeout to create a new EventSource after backoff
    - Increment reconnectAttempt
  - If `reconnectAttempt.current >= 5`: set error state "Connection lost. Refresh to check progress."
- On successful `message` or `progress` event, reset `reconnectAttempt.current = 0`

6. Add a `daily_limit_hit` field to the progress/completion detection. When status in a progress event is `"daily_limit_hit"`, treat it like completion but include a `dailyLimitHit: true` flag in CompletionData. Add `dailyLimitHit?: boolean` to `CompletionData` interface.
