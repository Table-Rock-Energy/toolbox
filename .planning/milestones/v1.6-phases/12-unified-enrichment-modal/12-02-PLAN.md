  - Active step label: e.g., "Step 2 of 3: Validating..."

- Step list: Each step shows circle indicator (numbered/spinner/check/error icon), step label ("Clean Up", "Validate", "Enrich"), and changes applied count on completion
- Step status icons: pending = gray number, active = Loader2 animate-spin in teal circle, completed = Check in green circle, error = AlertCircle in red circle, skipped = gray dash
- Error detail: If step has error, show error message in red text below step label
- Completion summary: Green bg-green-50 box showing total changes applied across all steps. E.g., "12 fields updated across 8 entries"
- Done button (only when pipelineStatus === 'completed' or 'error'): closes modal

**Create `frontend/src/components/UnifiedEnrichButton.tsx`:**

Props interface:
```typescript
interface UnifiedEnrichButtonProps {
  pipelineStatus: PipelineStatus
  entryCount: number
  anyStepEnabled: boolean
  onEnrich: () => void
  onReopen: () => void
  onUndo: () => void
  onClearHighlights: () => void
  hasChanges: boolean  // enrichmentChanges.size > 0
  hasSnapshot: boolean // preAutoApplySnapshot !== null
}
```

Button states per CONTEXT.md:
1. **Idle** (`pipelineStatus === 'idle'` and no changes): Teal gradient button (`bg-gradient-to-r from-tre-teal to-tre-teal/80`), Sparkles icon, label "Enrich (N)" where N = entryCount. Disabled if entryCount === 0 or !anyStepEnabled.
2. **Running** (`pipelineStatus === 'running'`): Same teal button, Loader2 icon with `animate-spin`, label "Running...". onClick calls `onReopen` (reopens modal).
3. **Completed** (`pipelineStatus === 'completed'`): Green-tinted button (`bg-green-100 text-green-800 border border-green-300`), Check icon, label "Enriched". onClick shows `window.confirm('Re-run enrichment? This will replace current results.')` then calls `onEnrich` if confirmed.
4. **Post-enrichment secondary actions** (rendered next to main button when `hasChanges || hasSnapshot`):
   - "Undo Enrichment" button: `RotateCcw` icon, gray outline style, calls `onUndo`. Only shown when `hasSnapshot` is true.
   - "Clear Highlights" button: `X` icon, gray outline style, calls `onClearHighlights`. Only shown when `hasChanges` is true.

Component renders a `<div className="flex items-center gap-2">` containing the main button and conditional secondary buttons.
