**Internal functions (all prefixed with `_`):**

1. `_extract_metadata(text: str) -> CaseMetadata` -- Extract from first page:
   - `county`: regex `r'(\w+)\s+COUNTY,\s+OKLAHOMA'`
   - `case_number`: regex for "CAUSE NO." spanning possible line break, capture the CD number (e.g., "CD 2026-000909-T"). Handle the case where "CAUSE NO. CD" is on one line and the number on the next.
   - `applicant`: regex `r'APPLICANT:\s+(.+?)(?:\n|$)'`
   - `legal_description`: regex for "SECTION(S)..." through "RANGE \d+ \w+"
   - `well_name`: regex `r'\(the\s+(.+?)\s+well\)'` from paragraph 2(D)
   - All fields Optional -- return None for any that don't match

2. `_strip_page_headers(text: str) -> str` -- Remove repeated headers/footers:
   - Page headers: lines matching "MULTIUNIT HORIZONTAL WELL" pattern
   - Exhibit A headers: lines matching `EXHIBIT "A"` or `EXHIBIT "A"` (smart quotes)
   - County/section summary lines that repeat on each page
   - Applicant name lines that repeat on each page
   - Page footers: "CASE CD CD2026-... PAGE X OF Y"
   - Standalone page numbers (single digit lines)
   - Use `re.MULTILINE` with `^...$` patterns. Be generous with stripping -- better to over-strip headers than leave them to corrupt entries.

3. `_extract_exhibit_a_section(text: str) -> str` -- Extract only the Exhibit A respondent list portion (after the application body, before any Exhibit B). Look for `EXHIBIT "A"` or `EXHIBIT\s+"?A"?` marker followed by "RESPONDENTS" or entry number 1.

4. `_split_into_sections(text: str) -> list[tuple[str, str]]` -- Split by section headers:
   - Headers (match LONGEST first to avoid substring ambiguity):
     - "CURATIVE RESPONDENTS WITH ADDRESS UNKNOWN:" -> "curative_unknown"
     - "RESPONDENTS WITH ADDRESS UNKNOWN:" -> "address_unknown"
     - "CURATIVE RESPONDENTS:" -> "curative"
     - "FOR INFORMATIONAL PURPOSES ONLY:" -> "informational"
   - Text before any header -> "regular"
   - Returns list of `(section_type, section_text)` tuples

5. `_parse_section_entries(text: str, section_type: str) -> list[PartyEntry]` -- Split text into entry blocks and parse each:
   - Split on entry number pattern: `r'\n(?=\d+\.\s+[A-Z])'` -- entry number followed by uppercase letter (distinguishes from street addresses like "12801 N Central")
   - For each block, call `_parse_entry_block()`

6. `_parse_entry_block(block: str, section_type: str) -> PartyEntry` -- Parse a single entry:
   - Extract entry number from first line: `r'^(\d+)\.\s+(.+)'`
   - Separate name lines from address lines. Address lines: start with digit (street number), "PO Box", or "P.O. Box". City/state/ZIP line: matches `r'[A-Z][a-z]+.*\s+[A-Z]{2}\s+\d{5}'`.
   - For "address_unknown" and "curative_unknown" sections: entries have no address lines
   - Handle "c/o" lines: extract into notes, treat remainder as address
   - Join name lines into full name text
   - Call `_classify_ecf_entity()` for entity type
   - Handle "now [name]" pattern BEFORE name parsing: extract maiden name to notes, rewrite name with married name
   - Strip "deceased", "possibly deceased", "Heirs and Devisees of" annotations from name before passing to name_parser
   - Call `parse_name()` from name_parser for individuals
   - Call `parse_address()` from shared/address_parser for address text
   - Set `flagged=True` if name is very short (<3 chars) or address is expected but missing (for regular/curative sections)

7. `_classify_ecf_entity(text: str) -> tuple[EntityType, list[str]]` -- Entity classification with ECF-specific patterns:
   - Check for "deceased" or "possibly deceased" (case-insensitive) -> EntityType.ESTATE
   - Check for "Heirs and Devisees of" -> EntityType.ESTATE
   - Fall back to `detect_entity_type()` from `app.utils.patterns` for Trust, LLC, Corporation, etc.
   - Return (entity_type, extra_notes_list)
