# Requirements: Table Rock TX Tools — GHL Prep Tool

**Defined:** 2026-02-25
**Core Value:** Any new data pipeline — from source to Mineral — can be built by defining only the unique parts (source adapter + transform logic) while inheriting everything else (UI, ingestion, enrichment, export).

## v1.1 Requirements

Requirements for GHL Prep tool. Each maps to roadmap phases.

### GHL Prep (Core)

- [x] **GHL-01**: User can upload a Mineral export CSV file
- [x] **GHL-02**: Tool title-cases all text fields (names, cities, counties, territories) using proper name handling (Mc/Mac/O' prefixes)
- [x] **GHL-03**: Tool extracts first campaign unit name from Campaigns JSON column and replaces column with plain text value
- [x] **GHL-04**: Tool adds "Phone" column mapped from "Phone 1" value as primary phone number
- [x] **GHL-05**: Tool ensures "Contact Owner" column exists (preserves if present, adds blank if missing)
- [x] **GHL-06**: User can preview transformed data in a table before exporting

### GHL Prep (Export)

- [x] **GHL-07**: User can download transformed CSV ready for GHL import

## v2 Requirements

Deferred to future release. Tracked but not in current roadmap.

### GHL Integration

- **GHL-08**: Direct API export into GoHighLevel (bypass CSV download)
- **GHL-09**: Contact Owner dropdown populated from GHL user list when no owner is set
- **GHL-10**: SmartList name assignment on import — assign contacts to a GHL smartlist
- **GHL-11**: Tag assignment on import — apply tags to contacts during GHL import

## Out of Scope

| Feature | Reason |
|---------|--------|
| Phone number formatting/validation | Risk breaking valid numbers, GHL handles natively |
| Data enrichment in GHL Prep | Belongs in Mineral CRM, not prep tool |
| Duplicate detection | GHL handles natively during import |
| Direct GHL API push (v1.1) | Build CSV export first, API integration in v2 |
