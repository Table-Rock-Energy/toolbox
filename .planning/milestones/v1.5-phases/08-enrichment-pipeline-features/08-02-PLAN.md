         - `enrich(tool: string, entries: object[], fieldMapping?: Record<string, string>): Promise<PipelineResponse>`

    2. Create `frontend/src/hooks/useEnrichmentPipeline.ts`:
       ```typescript
       interface UseEnrichmentPipelineOptions<T> {
         tool: string
         previewEntries: T[]
         updateEntries: (entries: T[]) => void
         editedFields: Map<string, Partial<T>>
         keyField: keyof T
         featureFlags: { cleanUpEnabled: boolean; validateEnabled: boolean; enrichEnabled: boolean }
       }
       ```
       Hook state:
       - `completedSteps: Set<'cleanup' | 'validate' | 'enrich'>` -- tracks which steps are done
       - `activeAction: 'cleanup' | 'validate' | 'enrich' | null` -- current processing step
       - `proposedChanges: ProposedChange[] | null` -- currently displayed proposals
       - `checkedIndices: Set<number>` -- which proposal indices are checked for apply (default: all checked)
       - `recentlyAppliedKeys: Set<string>` -- keys of entries that just got changes applied (for green highlight, cleared after 2s timeout)

       Sequential unlock logic (per CONTEXT.md decision):
       - `canCleanUp`: featureFlags.cleanUpEnabled && !isProcessing
       - `canValidate`: featureFlags.validateEnabled && (completedSteps.has('cleanup') || !featureFlags.cleanUpEnabled) && !isProcessing
       - `canEnrich`: featureFlags.enrichEnabled && (completedSteps.has('validate') || (!featureFlags.validateEnabled && (completedSteps.has('cleanup') || !featureFlags.cleanUpEnabled))) && !isProcessing

       onCleanUp/onValidate/onEnrich handlers:
       - If step already completed, show `window.confirm("Re-running will replace previous results. Manual edits are preserved. Continue?")` -- if no, return
       - Set activeAction, call pipelineClient.{step}(tool, previewEntries)
       - On success: set proposedChanges, checkedIndices = all indices, clear activeAction
       - On error: clear activeAction, log error

       onApply handler (per CONTEXT.md propose-review-apply decision):
       - Build updated entries array from current previewEntries
       - For each proposedChange where checkedIndices.has(its index in proposedChanges array):
         - Get the entry key via `String(previewEntries[change.entry_index][keyField])`
         - Check `editedFields.get(key)` -- if user edited this specific field AND change.authoritative is false, SKIP (user edits win)
         - If change.authoritative is true (Google Maps), apply regardless of user edits
         - Apply: `updatedEntries[change.entry_index] = { ...entry, [change.field]: change.proposed_value }`
       - Call updateEntries(updatedEntries)
       - Add affected entry keys to recentlyAppliedKeys, set 2s timeout to clear them
       - Mark step as completed in completedSteps
       - Clear proposedChanges

       onDismiss handler:
       - Clear proposedChanges without applying
       - Do NOT mark step as completed

       Return: { activeAction, proposedChanges, checkedIndices, completedSteps, recentlyAppliedKeys, canCleanUp, canValidate, canEnrich, isProcessing, onCleanUp, onValidate, onEnrich, onApply, onDismiss, toggleCheck (index: number) => void, toggleCheckAll () => void }

    3. Create `frontend/src/components/ProposedChangesPanel.tsx`:

       - Keep it simple -- no complex animations. Use Tailwind utility classes matching project conventions.

       - Preserve backward compatibility: if canValidate/canEnrich are undefined, use existing behavior (enabled when cleanUpEnabled/validateEnabled/enrichEnabled is true)
