    3. POST `/validate` endpoint:
       - Accept PipelineRequest body
       - Check `settings.use_google_maps` -- if False, return error
       - Define default field mapping per tool (from CONTEXT.md open question #1):
         ```python
         DEFAULT_FIELD_MAPPINGS = {
             "extract": {"street": "mailing_address", "street_2": "mailing_address_2", "city": "city", "state": "state", "zip": "zip_code"},
             "title": {"street": "address", "street_2": "address_2", "city": "city", "state": "state", "zip": "zip_code"},
             "proration": {"street": "address", "street_2": "address_2", "city": "city", "state": "state", "zip": "zip_code"},
             "revenue": {},  # Revenue has no address fields -- return empty
         }
         ```
       - Merge request.field_mapping over defaults (request overrides)
       - If no street field in mapping (e.g., revenue), return PipelineResponse(success=True, proposed_changes=[], entries_processed=len(entries))
       - Call `validate_addresses_batch()` in `asyncio.to_thread` with the resolved field names
       - Compare original vs validated entries to build ProposedChange list:
         - For each entry where address changed: create ProposedChange per changed field (source="google_maps", authoritative=True, confidence based on AddressValidationResult.confidence)
       - Important: Do NOT use validate_addresses_batch's auto-apply pattern. Instead, call `validate_address()` per entry and build ProposedChange objects from the diff. This lets the frontend control apply.

    4. POST `/enrich` endpoint:
       - Accept PipelineRequest body
       - Check enrichment enabled via `is_enrichment_enabled()` -- if False, return error
       - Build persons list from entries using field_mapping to find name/address/city/state/zip fields
       - Call `enrich_persons(persons)`
       - Transform EnrichedPerson results into ProposedChange list:
         - For each EnrichedPerson with phones: ProposedChange(field="phone", proposed_value=first phone number, source="pdl" or "searchbug" based on enrichment_sources)
         - For each with emails: ProposedChange(field="email", proposed_value=first email)
         - authoritative=False for all enrichment changes
       - Return PipelineResponse
