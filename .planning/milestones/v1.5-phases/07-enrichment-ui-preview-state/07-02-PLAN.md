  <action>
    Create `frontend/src/hooks/usePreviewState.ts` with these exports:

    ```typescript
    export interface UsePreviewStateOptions<T> {
      entries: T[]                     // Source entries from processing
      keyField: keyof T               // Unique identifier field
      flagField?: keyof T             // Optional field for flagged status (default: 'flagged')
    }

    export interface PreviewState<T> {
      // Core state
      previewEntries: T[]             // Entries with edits applied, flagged sorted to top
      excludedKeys: Set<string>       // Keys of excluded entries
      editedFields: Map<string, Partial<T>>  // Overrides per entry key

      // Derived
      entriesToExport: T[]            // Non-excluded entries with edits applied
      flaggedCount: number            // Count of flagged entries

      // Actions
      updateEntries: (entries: T[]) => void   // Replace all entries (after enrichment)
      toggleExclude: (key: string) => void
      toggleExcludeAll: () => void
      editField: (key: string, field: keyof T, value: unknown) => void
      resetEdits: () => void

      // Selection helpers
      isAllSelected: boolean
      isSomeSelected: boolean
      isExcluded: (key: string) => boolean
    }
    ```

    Implementation details:
    1. Accept entries via `options.entries`. When `entries` changes (reference), reset `editedFields` and `excludedKeys` (new data from processing/enrichment).
    2. `updateEntries`: Use a `useState` setter that replaces the entries array. This is for enrichment callbacks to push new data. Does NOT reset edits -- edits are keyed by stable key so they survive if the same entries come back with additional fields.
    3. `previewEntries` (useMemo): Apply edits from `editedFields` map, then sort with flagged entries first. Sort is stable: flagged first, then preserve original array order within groups. Use `flagField` (default `'flagged'`) to determine flag status.
    4. `entriesToExport` (useMemo): Filter `previewEntries` by `!excludedKeys.has(key)`.
    5. `toggleExclude(key)`: Add to set if not present, remove if present. Use functional state update.
    6. `toggleExcludeAll()`: If all visible are selected (none excluded), exclude all. If any excluded, clear exclusions.
    7. `editField(key, field, value)`: Update `editedFields` map. Merge with existing edits for that key.
    8. `resetEdits()`: Clear `editedFields` map.
    9. `isAllSelected`: `excludedKeys.size === 0`
    10. `isSomeSelected`: `excludedKeys.size > 0 && excludedKeys.size < previewEntries.length`
    11. `isExcluded(key)`: `excludedKeys.has(String(key))`

    Key constraint: All keys are coerced to `String()` for Set operations (handles numeric keys from entry_number).

    Pitfall avoidance (from RESEARCH.md):
    - Use stable key field, never array index
    - Functional state updates for exclusion toggles to avoid stale closures
    - `entriesToExport` derives from preview state, never from original entries
