## Tech Debt by Phase

### Phase 1: Auth Enforcement & CORS Lockdown
- **Pre-existing admin endpoint exposure:** Admin router excluded from router-level auth (correctly, for `check_user`), but 5 additional endpoints lack any auth: `GET /api/admin/users`, profile image upload/serve, preferences GET/PUT. These are not new — they were unprotected before this milestone.
- **CORS CI gap:** Production origin `https://tools.tablerocktx.com` not tested in CI suite (dev origin only). Low risk since production default is a literal in config.py.
- **Pre-existing:** `hasCSVData` unused variable in Proration.tsx (already cleaned in commit 78fc6e5)

### Phase 2: Encryption Hardening
- **Startup guard untested:** `SystemExit(1)` for missing ENCRYPTION_KEY in production cannot be exercised by current test setup (ASGITransport bypasses lifespan). A `TestClient`-based integration test would close this.

### Phase 3: Backend Test Suite
- **Parser coverage limited:** Only EnergyLink revenue parser has regression tests. Enverus and Energy Transfer parsers (the most complex) lack coverage.
- **CI Python version:** Workflow uses 3.13; docs specify 3.11+. Compatible but worth aligning.
- **History endpoint scoping:** Authenticated but not user-scoped — any authenticated user can view/delete any job (pre-existing authorization gap, tracked as AUTHZ-02 in future requirements).

### Total: 7 items across 3 phases
