  <action>
Create `backend/app/core/security_headers.py` with a Starlette `BaseHTTPMiddleware` subclass called `SecurityHeadersMiddleware`.

The `dispatch` method calls `await call_next(request)` then adds these 6 headers to the response before returning it:

1. `Content-Security-Policy` (SEC-01):
   ```
   default-src 'self'; script-src 'self'; style-src 'self' 'unsafe-inline' https://fonts.googleapis.com; font-src 'self' https://fonts.gstatic.com; img-src 'self' data:; connect-src 'self'; frame-ancestors 'none'; base-uri 'self'; form-action 'self'
   ```
   Note: `'unsafe-inline'` on style-src is required because React injects inline styles. `img-src` includes `data:` for inline SVGs. `frame-ancestors 'none'` supplements X-Frame-Options.

2. `Strict-Transport-Security` (SEC-02):
   ```
   max-age=31536000; includeSubDomains
   ```

3. `X-Frame-Options` (SEC-03):
   ```
   DENY
   ```

4. `X-Content-Type-Options` (SEC-04):
   ```
   nosniff
   ```

5. `Referrer-Policy` (SEC-05):
   ```
   strict-origin-when-cross-origin
   ```

6. `Permissions-Policy` (SEC-06):
   ```
   camera=(), microphone=(), geolocation=()
   ```

Then in `backend/app/main.py`, import and register the middleware AFTER the CORS middleware:
```python
from app.core.security_headers import SecurityHeadersMiddleware
app.add_middleware(SecurityHeadersMiddleware)
```

Place the `app.add_middleware(SecurityHeadersMiddleware)` line BEFORE the CORSMiddleware registration (since Starlette middleware is LIFO -- last added runs first, so security headers middleware added first means it runs after CORS, wrapping the response on the way out).
