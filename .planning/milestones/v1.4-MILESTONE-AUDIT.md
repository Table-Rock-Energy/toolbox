## Critical Gaps

### 1. EXP-03: case_metadata not sent in export request

**Location:** `frontend/src/pages/Extract.tsx` lines 455-460

The `handleExport()` function sends `entries`, `filename`, `county`, and `campaign_name` but does **not** include `case_metadata`. The backend `export_service.py` `_format_metadata_note()` generates a pipe-separated string for Notes/Comments from case_metadata, but it receives `None` because the frontend never sends it.

**Fix:** Add `case_metadata: activeJob?.result?.case_metadata` to the export request body.

**Impact:** County column works (passed separately). Notes/Comments loses legal description, applicant, and well name.

### 2. Phase 4: Missing VERIFICATION.md

Phase 4 (Frontend Integration) has no formal VERIFICATION.md. FE-01 through FE-04 are only verified by human observation in the summary (no programmatic evidence). The implementation exists and was visually confirmed, but the documentation gap means the 3-source cross-reference cannot fully validate these requirements.
